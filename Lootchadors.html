History:

Lootchadors is a game which was developed between 2017-2018 as part of a client brief for university. The project laid dormant between 2018-2021. During this time I constantly thought about new ideas for the game and starting the project again. In 2021 I quit my job and decided to make a go of trying to develop the game fully. Started from scratch and scrapped the original prototype.

About The Game:

Lootchadors is a local multiplayer game. The aim of the game is to loot museums and be the one to have the most money at the end of the round. You can throw each other, punch each other, use power ups on each other and dash into each other to cause mayhem!

The game is being developed in UE4 and I’m currently working towards having a polished prototype.

My Role:

For this project I’ve been doing gameplay programming, design, project management and managing contractors.

Some features I’ve designed and implemented:
  • Camera that moves its position and follows the average position between all players.
  • Offscreen Camera System which draws a view of a player if they are too small in the current viewport or offscreen. Means the main gameplay camera doesn’t zoom out so far that no one can see themselves in the game.
  • A Stat Effect system, players have a Stat Effect component manager which adds, updates and removes stat effects. Stat Effects are self contained and have their own logic which effects the player.
  • Strike combo system.
  • Grappling and throwing opponents.
  • Score Tracking.
  • Lootable objects & interaction system.
  • Picking up and throwing objects.
  • Taunting.
